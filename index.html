Register

User Misc
▼

Purpose: Register yourself with your payment information and email address.

/register payment_info: Method: Details email: your_email@example.com

Example:

/register payment_info: Paypal: user@example.com email: mystaffemail@gmail.com

Important Notes:

  • This command is the first step for all staff members.
  • Your payment_info must start with the method (e.g., Paypal:, Binance:, TRC20:, BEP-20:, Phone Cash:, Other:) followed by your address/ID/details.
  • Your email is used for access to shared resources (like Google Drive).
  • You must be registered to submit work and receive points/payments.
  • You can run this command again to update your information.

Submit

User Progress Points
▼

Purpose: Submit your completed work for a chapter or range of chapters, receive points, and trigger the next step in the workflow.

/submit channel: #series-channel chapter_range: number_or_range chapter_link: URL_to_work job: Job_Name

Examples:

/submit channel: #cool-manhwa chapter_range: 15 chapter_link: https://gdrive/link/to/ch15_TL job: Translation
/submit channel: #another-cool-manhwa chapter_range: 20-22 chapter_link: https://mega.nz/link/to/ch20-22_Edit job: Edit

Important Notes:

  • You must be registered (using the /register command) and have claimed the chapter step previously (or it must be the first step like Translation after Raw).
  • chapter_range: Can be a single number (e.g., 15) or a dash-separated range (e.g., 20-22).
  • chapter_link: A direct link (Google Drive, Mega, Mediafire etc.) where the next person can download your work. or the main drive folder if you're submitted several chapters at the same time.
  • job Select the job you completed: Translation, Proofreading, Cleaning, Typesetting, Quality Check, Edit (combines Cleaning & Typesetting), Q-Edit (combines Cleaning & Typesetting & Quality Checking). Managers and Raw Providers can use Raw.
  • You must have the corresponding Discord role for the job you are submitting.
  • Successful submission awards points (based on the series and step pay rates) and notifies the staff member responsible for the next step in the workflow.
  • The bot will prevent submission if someone else has already claimed or completed the step for that chapter.

Series Progress

General Progress
▼

Purpose: View the current progress status for all active chapters in a series channel.

/series_progress channel: #series-channel

Example:

/series_progress channel: #cool-manhwa

Important Notes:

  • Shows a visual progress bar (██ completed, ▄▄ ongoing, ░░ pending) for each avilable chapter.
  • Indicates the percentage complete and the next step needed (e.g., → PR).
  • If a step is ongoing, it shows who claimed it and since when.
  • Includes Claim buttons for the next available step on each chapter shown.
  • Use this command to find available work in a series and claim it to start working on it.
  • If there are many avilable chapters, the report will be paginated with Next/Previous buttons.

Claim

User Progress
▼

Purpose: Claim an available chapter step to start working on it.

Usage:

  • Click the corresponding button (e.g., "TL 05", "ED 12") on messages from triggered by /submit, /series_progress, or /chapter_progress commands.

Important Notes:

  • Claiming reserves the chapter/step for you, preventing others from claiming it.
  • The step status changes to "ongoing" in progress reports.
  • You generally cannot claim a new chapter/step if you already have one ongoing (unless you have an exempt role).
  • Users with a Restricted role cannot claim chapters.
  • Once clicked, the button becomes disabled.
  • After claiming, proceed with your work and use /submit when finished.

Undo

Admin Progress Points
▼

Purpose: Revert a progress step and/or submitted work entry for one or more chapters, optionally removing points.

/undo channel: #channel chapters: number_or_range job: Job_Name target: progress/work/both

Example:

/undo channel: #cool-manhwa chapters: 15 job: Translation target: Both

Important Notes:

  • Admin Only.
  • Use to correct mistakes in submissions or progress tracking.
  • job Select the job you want to undo: Raw, Translation, Proofreading, Cleaning, Typesetting, Quality Check, Edit (combines Cleaning & Typesetting), All (for All Steps).
  • target: progress removes only the claim/completion status; work removes the submission record and points; both does both.
  • This makes the step available again for claiming/submission.

Staff Request

Admin Progress Misc
▼

Purpose: Request substitute or new staff for one or more roles on a series, optionally automating access.

/staff_request role: RoleName [channel: #series-channel] [chapters_count: number] [number_of_candidates: number] [role_2: RoleName] [role_3: RoleName] [role_4: RoleName] [role_5: RoleName] [note: text] [permanent: Permanent/Temporary] [deadline: date/time_string] [urgency: Low/Medium/High/Urgent] [auto_access: Enabled/Disabled] [min_skill: A Tier/B Tier/C Tier]

Example:

/staff_request role: Korean translator channel: #cool-manhwa chapters_count: 3 role_2: Cleaner role_3: Typesetter permanent: Temporary urgency: High auto_access: Enabled min_skill: B Tier number_of_candidates: 2 deadline: in 3 hours note: Focus on ch 1-3

Important Notes:

  • Used by Admins and Managers to find staff for unavailable roles or new series needs.
  • You can request up to 5 different roles simultaneously (role, role_2...role_5).
  • Creates a request message in a dedicated channel (e.g., #staff-requests) mentioning the required role(s).
  • The message embed displays series info, points per chapter (fetched automatically), required roles, chapter count (default 1), deadline/urgency, and optional manager note.
  • channel: Defaults to the channel where the command is run if not specified.
  • permanent: Indicates if the need is long-term (Permanent) or short-term (Temporary, default).
  • deadline: Accepts flexible date/time strings (e.g., "tomorrow 5pm", "in 2 days").
  • urgency: Sets priority (Low, Medium, High, Urgent). Default is Low.
  • Staff Interaction: Users with the required Discord role(s) can click buttons on the request message to show/withdraw interest. The embed updates to show interested candidates per role.
  • auto_access: Enabled: Attempts to automatically grant channel and Google Drive access to interested users.
    • Requires: The series must have a valid Google Drive link registered via /series details, the bot must have 'Editor' permissions on that folder, and the interested user must have their email registered with the bot.
    • Grants access only if the user has the base Discord role (e.g., Proofreader).
    • min_skill: If set (A/B/C Tier), only users with that skill tier role (or higher) will get auto-access.
    • number_of_candidates: (Default 1) Limits how many users per role are granted auto-access. Others can still show interest but won't get automatic access.
    • Users are notified in the series channel upon successful access grant.
  • The command invoker is notified briefly when someone shows/withdraws interest.
  • Dependencies: Requires series points and Google Drive links to be configured using /series details for full functionality (points display, auto-access). Requires users to have registered emails for Drive auto-access.

Warn

Admin Points Misc
▼

Purpose: Allows Admins to issue a formal warning to a user, optionally deducting points and/or restricting their ability to work.

/warn target: @user reason: reason_text [points: number] [restrict: True/False]

Example:

/warn target: @LazyUser reason: Missed deadline for Proofreading Chapter 10 of #cool-manhwa without notice. points: 10 restrict: True

Important Notes:

  • Admin Only.
  • Logs the warning with the reason and timestamp.
  • points: If specified, deducts this amount from the user's balance.
  • restrict: True: Assigns a Restrict Work role to the user, preventing them from claiming new tasks via buttons.
  • An announcement is sent to a specific staff channel, and the user is typically DMed.
  • Accumulating multiple active warnings (e.g., 3+) puts the user at risk of being fired.

User Chapters

General Admin User Progress Points
▼

Purpose: View a list of chapters submitted by yourself or another user, with options to filter by series or job type.

/user_chapters [member: @user] [series: #series-channel] [job: Job_Name]

Examples:

/user_chapters
/user_chapters member: @SomeUser
/user_chapters series: #cool-manhwa job: Editing

Important Notes:

  • Displays a list of submitted chapters for the specified user (defaults to yourself).
  • Regular users can only view their own submissions.
  • Admins can view submissions for any member.
  • Optional Filters:
    • series: Filters the list to show only submissions for the specified series channel.
    • job: Filters the list to show only submissions matching the selected job type (Translation, Proofreading, Editing, Quality Checking).
  • The output shows each chapter name (e.g., #channel-name chapter_number) and lists the final job(s) performed for that chapter along with the relative submission date (e.g., Editing - an hour ago).
  • Chapters are sorted by the earliest submission date associated with them.
  • If the list contains more than 15 chapters, it uses pagination with Previous/Next buttons.

Series Rates

Admin Points Misc
▼

Purpose: Allows Admins to set or update the points awarded per job for a specific series.

/series rates channel: #channel [translating_points: num] [proofreading_points: num] [editing_points: num] [qc_points: num] [speed_run: Yes/No]

  • speed_run: Marks the series status as Speed Run or not. The default is Not Speed Run.

Series Deadlines

Admin Progress Misc
▼

Purpose: Allows Admins to set or update the expected completion time (deadline) for each job step on a series.

/series deadlines channel: #channel [translation: time] [proofreading: time] [editing: time] [quality_checking: time]

Example:

/series deadlines channel: #cool-manhwa translation: 2h proofreading: 45m editing: 1h30m quality_checking: 30m

Important Notes:

  • Admin Only. Part of the /series command group.
  • Specify time using h for hours and m for minutes (e.g., 2h, 90m). Defaults may apply if not set:
    • Translation: 45 Minutes
    • Proofreading: 20 Minutes
    • Editing: 45 Minutes
    • Quality Checking: 20 Minutes
  • These deadlines are used by the bot's reminder system to notify staff and admins about overdue tasks claimed via claim buttons.
